# Project Workflow

## Scope

Apply this workflow to general tasks in this project.

## Execution Rules

1. Understand the user intent and act directly unless clarification is required.
2. Prefer minimal, targeted changes over broad refactors.
3. Verify outcomes with the smallest relevant check.
4. Do not revert unrelated local changes.
5. Report changed files and concrete results.

## Validation

- For file updates, run a focused sanity check.
- If checks cannot run, state why and provide next verification step.

## Response Style

- Keep responses concise and action-oriented.
- Include command output highlights when user asks for them.
- Offer one practical next step when useful.
