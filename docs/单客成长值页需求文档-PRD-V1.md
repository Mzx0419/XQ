# 单客成长值页 · 需求与设计说明（PRD V1）

> **路由**：`private-ops-detail`（实现用；还原静态页时可理解为「顾客详情内的会员成长一页」）  
> **版本**：V1  
> **文档性质**：本文档以**还原静态页面与交互**为目标，描述**版式层级、模块顺序、可见文案结构、状态与强调规则**。技术字段与枚举若需开发对接，见文末 **附录**。

---

## 1. 产品一句话

美容师 / 顾问在接待前打开本页，在**约 10 秒内**回答三件事：**这人值不值得重点运营**、**行为偏哪类**、**有没有券能拿来促预约**。页面遵循 **结论在上、证据在下、数字少而准**。

---

## 2. 谁在什么场景用

| 维度 | 说明 |
| --- | --- |
| 使用者 | 美容师、顾问（时间碎、信息要一眼懂） |
| 典型场景 | 接待前扫一眼定沟通策略；复购/召回时结合券到期；识别愿分享、愿裂变的顾客 |

---

## 3. 整页结构（自上而下阅读顺序）

页面为**单列纵向流**，按顺序固定为五块。**不要**在首屏把「券」的核心数字硬塞进最顶上的「价值区」——券的抓手在 **区块 C** 完整呈现。

```text
┌─────────────────────────────────────┐
│  导航/顶栏（可选文案见 §6）              │
├─────────────────────────────────────┤
│  A 价值结论 + 成长值摘要（首屏核心）      │
├─────────────────────────────────────┤
│  B 行为画像（五维徽章 + 任务偏好）       │
├─────────────────────────────────────┤
│  C 优惠券与促预约（不放成长值累计）      │
├─────────────────────────────────────┤
│  D 今日沟通建议（可直接照读）            │
├─────────────────────────────────────┤
│  E 最近关键动态（少量，可展开）          │
└─────────────────────────────────────┘
```

---

## 4. 区块 A：顾客价值总评 & 成长值概览

### 4.1 设计意图

- **先给结论**：第一眼是「价值标签」+「一句话原因」，再是等级与成长值数字。  
- **成长值只在本区出现一次**：当前剩余、累计获得、累计兑换；**不要在区块 C 再重复这三类数字**（避免同一页两处口径打架）。

### 4.2 版式层级（从上到下）

1. **价值标签区（主视觉）**  
   - **标签**：四选一的中文结论（对应业务含义见下表），需像「状态胶囊」或「强调标题」呈现，一眼可辨。  
   - **原因**：紧跟一行短句（建议不超过约 **16 字**），解释「为什么是这类顾客」，与标签**同时出现**，不可只显示标签不显示原因。

| 展示用中文（示例） | 含义（给设计/文案统一用词） |
| --- | --- |
| 高价值 | 值得重点维护的高贡献顾客 |
| 有潜力 | 当前一般但上升明显，可加码运营 |
| 待挽回 | 有流失风险或需主动拉回 |
| 普通 | 默认档，无特殊强调 |

2. **等级标识**  
   - **仅文字**：如「V5」或产品内统一的中文等级名，**小标签**样式即可；不要求与 App 会员中心同款的独立等级插画。

3. **成长值摘要区（数字区）**  
   - 三个并列指标，文案建议固定为：  
     - **当前剩余**（可兑权益的成长值余额）  
     - **累计获得**  
     - **累计兑换**（或「已消耗/已兑换」，与产品统一）  
   - 若首屏高度紧张，允许将「累计获得 + 累计兑换」**合并为一行说明句**（如「累计获得 xxx，已兑 xxx」），但**当前剩余**建议始终保留为独立数字。

4. **可选辅助（弱一级）**  
   - 近 30 天到店次数、近 90 天成长值趋势（用「上升 / 持平 / 下滑」或箭头，**不额外堆阿拉伯数字**）。可与 B/D 对齐摆放，本区可省略。

### 4.3 状态与文案（价值标签）

- 数据不足或新客：**标签**仍给「普通」类；**原因句**用固定友好短句（如「数据积累中，建议到店后关注任务与券」），**禁止整行空白**。  
- 价值判定与数据如何映射：**业务规则待数据/运营定稿**；设计上预留四类标签样式即可。细粒度「这人爱分享还是爱活跃」以 **区块 B** 为准（结论在 A，证据在 B）。

## 5. 区块 B：行为画像

### 5.1 设计意图

用**固定五条**维度，让顾问快速贴上「行为类型」：活跃、交互、链接（权益/活动）、分享、裂变。每条对应 **徽章档位**：无 → 铜 → 银 → 金（与会员体系一致）。

### 5.2 版式建议

- **默认**：五条纵向排列或五格矩阵（以设计稿为准）；每条包含：**维度名称** + **档位（铜/银/金/未获得）** + 可选「近 30 天简要状态」（如高/中/低活跃）+ 可选小数字「该类任务完成次数」。  
- **折叠区「任务偏好」**：标题如「他常做哪些任务」；默认收起，点击「展开偏好」后最多展示 **3 条**（任务名 + 近 30 天次数）。避免首屏被任务列表撑满。

### 5.3 五维名称（展示文案）

高活跃（活跃分子）、高交互、高链接、高分享、高裂变——与内部命名对齐即可，用户侧语言要通俗。

---

## 6. 区块 C：优惠券与促预约

### 6.1 设计意图

回答：**爱不爱兑券、爱不爱到店用券、有没有快过期能催预约**。  
**本区不出现**「成长值余额 / 累计获得 / 累计兑换」三块数字（已在 A 展示）。

### 6.2 信息优先级（从上到下或视觉权重）

1. **强提醒**：即将过期券数量（如 7 天 / 15 天 / 30 天内）——适合 **红/橙** 强调。  
2. **当前未使用券数量**。  
3. **近 30 天「用券且到店」类次数**（促到店证据）。  
4. 再往下：累计获券、累计兑换、累计使用等可作次要信息或一行汇总。

### 6.3 促预约提示

- 一行 **行动导向短句**（建议约 **20 字内**），例如：「7 天内有 2 张服务券到期，建议催预约」「爱兑换但用得少，可引导到店核销」。  
- 可与全局「话术区」结合（见 §9）。

---

## 7. 区块 D：今日沟通建议

### 7.1 设计意图

把「券到期风险 + 到店引导动作」转换为**可直接复述的话术**，减少一线临场组织语言的成本。

### 7.2 版式与交互

- 卡片结构：标题（今日沟通建议）+ 一句主话术 + 一行弱提示。  
- 右上提供「复制话术」按钮，点击后用于快速复述/转发。  
- 文案长度建议 30~45 字，避免过长导致首屏阅读负担。

### 7.3 口径来源

- 由区块 C 的到期券与用券行为生成，优先引用「7天内到期」与「当前未使用」信号。  
- 无强风险时可回退为温和提醒话术（如“可结合本次项目推荐可用券”）。

---

## 8. 区块 E：最近关键动态

### 8.1 设计意图

给**证据感**，避免长列表。每条：**时间** + **标题** + **一行补充**（如「+20 成长值」「到店核销」）。

### 8.2 交互

- 默认 **3 条**；「更多」展开至最多 **5 条**；再要看全量 → **跳转**独立流水页（本页不承载全量明细）。

### 8.3 事件类型（设计时需区分图标或标签色）

任务完成、券兑换、券使用、成长值过期等——用轻量区分即可，不要做成复杂时间轴图表。

---

## 9. 全局界面元素（可选增强）

- **顶栏标题**：偏人话，如「顾客会员一览」，弱化「成长值/中台」等技术感。  
- **话术条（可选）**：浅底卡片，标题如「今天可以这样说」，内文由促预约结论改写为**可直接照读**的一句话（常结合区块 C）。  
- **行为画像压缩态（可选）**：在 B 区顶部增加一行「人话总结」+ 徽章缩写（如「活跃金 · 交互银 · …」），与详细五行二选一或并存由稿定。

---

## 10. 全局视觉与文案原则

| 原则 | 说明 |
| --- | --- |
| 结论优先 | 每块先见人话结论，数字佐证 |
| 密度控制 | 除 A 区成长值三项（或合并一句）外，其余区块仍以少量关键数字为主 |
| 过期强提醒 | 快过期券用红/橙体系，引导「立刻约到店」 |
| 可解释 | 价值标签必须带「为什么」的一句原因 |
| 明确不做 | 默认不出现复杂图表；不堆点赞/浏览等细碎行为；不做群体统计 |

---

## 11. 体验验收（非技术）

使用者 **10 秒内**应能口头说出：

1. 这位顾客是偏 **高价值 / 有潜力 / 待挽回 / 普通** 哪一类，且**知道一句理由**。  
2. 行为上偏 **哪类**（五维里哪几项突出）。  
3. **有没有**快过期或可用的券、**能不能**用来促预约。
4. 可以直接复述一条系统建议话术（无需现场临时组织句子）。

**分区可见性**：价值与成长值核心在 **A** 区首屏可读；券的关键提示在滑到 **C** 时清晰可见，**不要求**把券数字塞进 A。
