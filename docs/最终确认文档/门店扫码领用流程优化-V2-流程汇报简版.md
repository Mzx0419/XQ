# 门店扫码领用流程优化 V2

## 1. 目标

在不明显影响结算效率的前提下，提升扫码合规率，降低库存损耗，并实现异常可追溯。

## 2. 对比流程

### 员工接待顾客产品领用流程图

```mermaid
flowchart TD
    A[员工接待顾客做服务] --> B{第一步}
    B --> C[直接开单]
    B --> D[直接领用产品]

    C --> E[选择服务内容并提交订单]
    E --> F[系统提示: 需要扫服务产品码]
    F --> G{是否现在扫码}
    G --> H[现在扫码]
    G --> I[选择稍后再扫]

    H --> J[前台结算]
    I --> J
    D --> E

    J --> K[顾客扫码]
    K --> L{订单是否有扫过产品码}
    L --> M[是: 可以复核(展示服务产品内容)]
    L --> N[否: 提示：未扫产品]
```



### A流程：强绑定审核（财务观点）

- 技师提交结算后提示扫码；可先跳过，进入前台审核。
- 存在未扫描项时，必须前台审核后才可放行结算。
- 前台复核界面增加“服务产品内容/产品使用信息”区，至少包含：产品名称、规格、应扫数量、实扫数量、本次用量、差异项。
- 审核结果留痕（通过/驳回、备注、操作人、时间）。
- 即使审核通过，未扫描订单仍不计技师业绩。

强制流程目的（记录）：

- 人员介入可更强控制使用成本（效果最好，但流程更死板）。
- 覆盖“顾客先结账”等特殊场景，避免随意扫码无核实。
- 保持业绩约束不放松：未扫描即不计业绩。

### B流程：最简开发+数据复盘治理

- 保持现有主流程，不新增前台审核链路。
- 支持“无服务单先领用”，后续补关联服务单。
- 不计业绩规则；未绑定服务单不会显示到业绩复核功能中。
- 前台复核界面增加“服务产品内容/产品使用信息”区，至少包含：产品名称、规格、应扫数量、实扫数量、本次用量、差异项。
- 通过数据复盘做针对性治理：按“项目-产品”重叠比对，结合技师/门店维度，对异常订单或异常产品预警。

## 3. 会议确认问题（重点）

1. 本期采用A还是B？若先B，什么阈值触发升级到A？
2. A流程边界：哪些场景必须前台审核？审核通过/驳回标准是什么？
3. 业绩口径：未扫描订单按整单还是按项目不计业绩？退单/反结算是否冲销？
4. B流程边界：无服务单先领用的允许条件、补关联时限、超时责任归属。
5. 数据治理口径：重叠比对周期、技师/门店预警阈值、按订单还是按产品触发预警。

## 4. 责任分工

- 技师：扫码、先领用补关联、异常说明。
- 前台：A流程审核放行并留痕。
- 店长：B流程预警跟进与闭环。
- 财务：强绑定口径与业绩规则确认、月度复盘。

## 5. 指标与验收

- 指标：强绑定审核通过率、未绑定提醒触达率、不计业绩命中率、异常预警命中率。
- 验收：流程可执行、口径可对账、盘点差异趋势改善（上线4周观察）。
