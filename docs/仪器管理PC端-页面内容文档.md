# 仪器管理PC端-页面内容文档

## 1. 文档目标

- 面向库管角色，定义「仪器管理列表页」与「添加仪器页」的页面内容、交互与状态规范。
- 保障核心字段与业务规则一致，支持直接进入UI设计与前端实现。

## 2. 适用角色与核心任务

- 角色：库管（主）、店长/运营（查看为辅）。
- 核心任务：
  - 快速查看各门店仪器档案与状态。
  - 按条件检索仪器并执行查看、编辑、启用/停用。
  - 新增仪器并完成唯一二维码绑定。

## 3. 信息架构总览

- 一级入口：库存管理 > 仪器管理
- 二级页面：
  - 页面A：仪器管理（列表）
  - 页面B：添加仪器（表单）

## 4. 页面A：仪器管理列表页

### 4.1 页面目标

- 提供仪器档案全量可视化与快速检索。
- 支持状态管理与常见维护操作。

### 4.2 顶部区域与筛选区

- 页面标题：仪器管理
- 说明文案：统一管理仪器档案、门店归属、有效期与二维码绑定关系。
- 按钮：
  - 新增仪器（主按钮）
  - 导出（次按钮，可选）
- 筛选组件（从左到右）：
  - 关键词搜索：支持仪器ID（系统ID）/机器生产编号/名称/型号/二维码编码
  - 所属门店：单选下拉
  - 当前状态：单选下拉（未启用、在店、返厂维修、报废）
  - 类型：单选下拉（发数、片数）
  - 有效期状态：单选下拉（全部、正常、临期、已过期）
  - 重置筛选（文本按钮）

### 4.3 表格字段定义


| 字段      | 展示规则                      | 说明            |
| ------- | ------------------------- | ------------- |
| 仪器ID    | 固定宽度，支持复制                 | 系统唯一标识        |
| 机器生产编号  | 常规文本，支持复制                 | 厂商出厂编号（非系统ID） |
| 名称      | 超长省略，悬浮显示全称               | 业务名称          |
| 型号      | 常规文本                      | 厂商型号          |
| 类型      | Tag样式：发数/片数               | 决定后续使用单位      |
| 累计使用次数  | 数值展示                      | 统计该仪器历史累计使用量  |
| 有效期     | `YYYY-MM-DD`；临期/过期高亮      | 资产有效截止日期      |
| 所属门店    | 门店名称                      | 当前归属门店        |
| 当前状态    | Tag样式：未启用/在店-门店名/返厂维修/报废  | 返厂维修状态不可用     |
| 唯一识别二维码 | 显示编码摘要 + 查看按钮             | 与仪器ID一一对应     |
| 最近更新时间  | `YYYY-MM-DD HH:mm`        | 便于追溯          |
| 操作      | 查看、编辑、使用记录、转店记录、二维码、启用/停用 | 按权限显示         |


### 4.4 状态Tag与颜色规范

- 未启用：中性态（灰蓝系标签）
- 在店：成功态（绿色系标签，文字深色），展示为“在店-门店名称”
- 返厂维修：警示态（橙色系标签）
- 报废：禁用态（灰色系标签）
- 临期：在有效期字段使用橙色提示（例如“7天后到期”）
- 已过期：在有效期字段使用红橙提示（例如“已过期12天”）

### 4.5 操作列交互

- 操作入口：每行直接展示「查看」「编辑」，其余动作收敛到「更多」按钮；点击「更多」后以小弹窗菜单展示。
- 查看：进入详情抽屉/详情页（只读）。
- 编辑：进入编辑页，允许修改非唯一性字段。
- 使用记录：查看该仪器按订单维度的历史使用记录与读数变化。
- 转店记录：查看该仪器历史门店流转记录与时间线。
- 二维码：打开二维码弹窗，展示二维码状态与打印操作。
  - 未打印过：显示“打印二维码”主按钮。
  - 已打印过：显示“补打”按钮，并显示最近打印时间与操作人。
- 停用：当状态为“在店”时显示“停用”，二次确认后状态切换为“报废/停用”（按业务配置）。
  - 停用后结果：
    - 该仪器不可在领用、扫码登记等业务中被选择。
    - 列表状态更新为“报废/停用”，默认不允许转店与补打操作。
    - 历史使用记录与转店记录继续保留，仅可查看不可修改。
- 启用：当状态为“返厂维修/报废/未启用”时显示“启用”，确认后恢复可用状态（通常转为“在店-所属门店”或“未启用”，按业务配置）。
- 不可操作限制：
  - 返厂维修中的仪器允许查看与编辑基础信息，不允许执行可用性相关操作。

### 4.6 列表页异常与边界状态

- 加载态：表格骨架屏。
- 空数据态：展示“暂无仪器数据”，提供“新增仪器”入口。
- 无筛选结果：展示“未找到匹配仪器”，提供“清空筛选条件”。
- 请求失败：展示错误提示与“重试”按钮。
- 无权限：显示无权限占位并引导联系管理员。

## 5. 页面B：添加仪器页

### 5.1 页面目标

- 快速、准确创建仪器档案并完成二维码绑定。
- 在提交前拦截关键错误，保证数据一致性。

### 5.2 页面结构（分组）

1. 基础信息
2. 归属与状态
3. 有效期与二维码信息
4. 底部操作区

### 5.3 字段清单与规则


| 分组        | 字段        | 类型            | 必填    | 规则                                                    |
| --------- | --------- | ------------- | ----- | ----------------------------------------------------- |
| 基础信息      | 仪器ID      | 系统生成/只读       | 是     | 保存时生成，不可重复                                            |
| 基础信息      | 机器生产编号    | 文本输入          | 是     | 厂商出厂编号，支持字母/数字/中划线，1-50字，不可重复                         |
| 基础信息      | 仪器名称      | 文本输入          | 是     | 2-50字                                                 |
| 基础信息      | 型号        | 文本输入          | 是     | 1-50字                                                 |
| 基础信息      | 类型        | 单选            | 是     | 发数/片数                                                 |
| 基础信息      | 初始次数/初始片数 | 数字输入          | 是     | 类型=发数时填写初始次数；类型=片数时填写初始片数；整数且>=0                      |
| 基础信息      | 平均一次使用区间  | 区间输入（最小值~最大值） | 是     | 类型=发数时填写平均一次使用发数区间；类型=片数时填写平均一次使用片数区间；最小值<=最大值且均为整数>0 |
| 归属与状态     | 所属门店      | 下拉选择          | 条件必填  | 仅当当前状态=在店时展示并必填                                       |
| 归属与状态     | 当前状态      | 单选            | 是     | 默认“未启用”                                               |
| 有效期与二维码信息 | 有效期       | 日期选择          | 是     | 不早于入库日期；独立维护                                          |
| 有效期与二维码信息 | 唯一识别二维码   | 系统生成（只读）      | 保存后生成 | 创建时不可编辑，保存后可打印二维码标签                                   |
| 有效期与二维码绑定 | 备注        | 多行文本          | 否     | 0-200字                                                |


### 5.4 二维码与有效期规则（重点）

- 创建阶段不需要二维码录入操作。
- 保存成功后由系统自动生成二维码编码，并提供“打印二维码”操作。
- 仪器ID与唯一识别二维码可使用同一编码（允许相同值）。
- 机器生产编号用于记录设备实体出厂标识，与系统生成的仪器ID并存。
- 有效期与二维码为独立字段，二者无绑定关系。
- 二维码编码作为唯一标识时全局不可重复绑定。
- 补打二维码默认沿用原编码，仅补打标签；若确需换码，必须记录换码原因并保留新旧码映射。

### 5.5 二维码弹窗与打印规则（新增）

- 入口：列表页操作列「二维码」。
- 弹窗字段：
  - 仪器ID（只读）
  - 二维码编码（只读）
  - 打印状态（未打印过 / 已打印过）
  - 最近打印信息（打印时间、操作人；仅已打印过显示）
- 按钮规则：
  - 未打印过：不展示二维码图片与下载按钮，仅显示“生成并打印二维码”。
  - 已打印过：显示“补打”。
- 页面进入规则：
  - 点击操作列“二维码”后，系统根据该仪器打印历史自动进入对应页面（未打印页或已打印页），用户无需手动选择。
- 提交结果：
  - 成功：提示“二维码打印任务已创建”，写入操作日志。
  - 失败：提示失败原因并保留上下文信息。
- 日志要求：记录仪器ID、二维码编码、操作类型（首打/补打）、操作人、时间、所属门店。
- 返厂再入库沿用原二维码，不重新生成。
- 若仪器报废，保留历史二维码映射关系，仅变更状态。

### 5.5 表单校验与报错文案

- 必填缺失：请完善必填信息后提交。
- 名称超长：仪器名称最多50个字符。
- 有效期非法：有效期不能早于入库日期。
- 初始值非法：初始次数/片数需为大于等于0的整数。
- 平均值非法：平均一次使用区间需满足最小值<=最大值，且两者均为大于0的整数。
- 二维码重复：该二维码已绑定其他仪器，请核对后重试。
- 状态提醒：当前状态为返厂维修，该仪器不可被领用或扫码登记。

### 5.6 提交与返回逻辑

- 主按钮：保存并返回列表
- 次按钮：取消
- 提交成功：提示“仪器已添加，二维码已生成”，支持立即打印二维码或返回列表并高亮新记录。
- 提交失败：保留已填内容，定位首个报错项。

## 6. 页面文案建议

- 列表标题：仪器管理
- 列表副标题：查看并维护仪器档案、状态与二维码绑定信息
- 新增按钮：新增仪器
- 表单标题：添加仪器
- 表单副标题：填写基础信息并完成二维码唯一绑定
- 停用确认标题：确认停用该仪器？
- 停用确认说明：停用后该仪器将不可用于领用与扫码登记。

## 7. 视觉与布局规范（PC）

- 布局：顶部标题与操作区 + 卡片式筛选区 + 表格区。
- 建议宽度：内容区最小1200px，表格横向滚动兜底。
- 颜色规范（使用项目统一色板）：
  - 主色：`#2563EB`
  - 主文字：`#1F2937`
  - 次文字：`#64748B`
  - 背景：`#F5F7FB` / `#FFFFFF`
  - 边框：`#E2E8F0`
  - 警示：`#F59E0B`
- 交互规范：
  - 当前激活筛选项仅允许一个激活态样式；切换后清除上一个激活残留。
  - 按钮、输入框、下拉、Tag采用统一圆角和间距。

## 8. 验收清单

- 列表页已覆盖核心字段与操作闭环。
- 添加页已覆盖必填、唯一性、状态可用性校验。
- 二维码与仪器ID的一一对应规则明确且前后一致。
- 空态、失败态、无权限态均有明确页面反馈。
