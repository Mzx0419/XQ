# 成长值 PRD 模板套件（填写说明）

## 使用目标
- 本套件用于产出可直接驱动 AI 生成静态原型的 PRD。
- 交付采用“主文档 + 子文档链接”结构，支持多人并行评审。

## 文档目录
- `01-PRD总览.md`
- `A-页面信息架构与模块定义.md`
- `B-交互与业务规则.md`
- `C-数据口径与算法规则.md`
- `D-视觉与样式约束.md`
- `E-埋点与验收.md`

## 强制规则（不满足则不可进入开发）
- 目标必须可衡量：包含指标、目标值、观察窗口。
- 页面必须可生成：包含组件清单、状态清单、文案与交互约束。
- 规则必须可执行：触发条件、边界、异常、降级、频控不可缺失。
- 验收必须可测试：每条验收都要有输入条件与预期结果。

## 写作禁用词
- 禁用：可能、适当、尽量、尽快、体验更好、显著提升。
- 替换为：明确阈值、时间窗口、责任方、判定口径。

## 评审顺序建议
1. 先过 `01-PRD总览.md`（目标/范围/优先级）。
2. 再过 `C-数据口径与算法规则.md`（避免后续返工）。
3. 再过 `A` 与 `B`（页面与交互）。
4. 最后过 `D` 与 `E`（样式与验收埋点）。
