# PRD 总览（技师首页优化-草图版）

## 1. 背景与问题定义
- 业务背景：技师首页承载日常执行入口，当前首页对一线技师“先做什么”指引不够聚焦。
- 设计基线：`设计/成长值.pen` 中的 `店务首页-1比1复刻` 为当前线上形态映射，本次为增量优化，不做整页重构。
- 当前问题（草图版聚焦可视层）：
  1. 首页主区信息分散，快捷操作与关键经营数据关联弱，且缺少销售人数/服务人数展示。
  2. 底部栏（店务/顾客/应用/数据）入口语义不统一，任务导向不明确。
  3. 私域运营入口与成长值动作链路衔接不直观。
- 本次仅使用的可观测链路：`首页曝光 -> 入口点击 -> 目标页到达`

## 2. 目标（草图级）
| 目标项 | 当前值 | 目标值 | 观察窗口 | 责任角色 |
|---|---:|---:|---|---|
| 首页主区关键动作点击率 | 待补充 | 提升（评审后补数值） | 7天 | 产品/设计 |
| 底部栏入口点击占比 | 待补充 | 结构优化后可识别提升 | 7天 | 产品 |
| 私域分享入口可达率 | 待补充 | 入口可达且一跳到位 | 7天 | 产品/前端 |

## 3. 用户与场景
- 目标用户：美容师（非管理看板）
- 核心场景（按优先级）：
  1. 在首页快速完成店务核心动作（快捷操作不变 + 数据概览增强）。
  2. 查看“今日待到店”并按到店时间提前准备接待，执行确认到店/取消预约。
  3. 通过底部栏进入顾客运营（产值目标/预约达标/跟进场景，含未评价与产值进度迁移）。
  3. 通过应用入口执行私域运营（查看成长值并直接分享）。

## 4. 范围边界（MVP）
- 本轮范围（做）：
  - 四块并行出草图：店务、顾客运营、私域运营、数据。
  - 首页主焦点仅做店务主区（快捷操作不动 + 数据概览补充销售人数/服务人数 + 今日待到店优化）。
  - 顾客/应用/数据主要优化底部栏入口与跳转语义（底部栏新增私域模块）。
  - 今日服务单未评价、产值进度迁移至顾客运营模块。
  - 以 `店务首页-1比1复刻` 为母版，输出差异化改动草图（新增/调整/保留）。
- 非本轮范围（不做）：
  - 不做完整业务流程改造，不做深交互状态机。
  - 不做数据模块业务细化，仅保留入口与占位页。
  - 不做完整埋点上线方案，仅定义草图评审检查项。

## 8. 对 AI 生成原型的输入摘要（复制即用）
- 页面名称：技师首页优化（草图版）
- 设备规格（宽高）：390 * 844（移动端）
- 区块顺序：Header -> 店务数据概览（含销售人数/服务人数）-> 快捷功能（保留）-> 今日待到店（时间排序）-> 今日未传服务图（保留）-> 底部栏（店务/顾客/应用/私域/数据）
- 每个区块组件清单：标题栏、数据概览卡、快捷功能宫格、待到店列表卡、底部导航栏、占位页卡片
- 每个区块状态：默认态必须；其余状态仅对占位页保留空态和异常态
- 交互动作与跳转：今日待到店支持确认到店/取消预约；点击底部栏进入对应模块页；私域页保留成长值分享按钮
- 文案与样式约束：快捷功能和今日未传服务图保持原样；数据模块统一使用“建设中”占位文案
