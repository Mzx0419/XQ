## 3. 区块定义（逐块必填）
### 区块：`上方数据概览区`
- 目标：在首页顶部直观看到核心经营人数与业绩表现
- 组件清单：
  - 组件名：数据概览卡组
  - 类型（卡片/列表/按钮/分段/弹层）：卡片
  - 字段：业绩、到店人数、消费人数、销售人数、服务人数
- 数据来源字段：performanceValue、arrivalCount、consumeCount、salesCount、serviceCount
- 可见条件：首页默认可见
- 排序规则：业绩 > 到店人数 > 消费人数 > 销售人数 > 服务人数
- 文案约束：指标名固定，不做营销文案
- 跳转行为：点击指标进入对应统计占位页
- 变更类型：调整（在现有数据概览上新增销售人数/服务人数）

### 区块：`店务快捷功能区`
- 目标：让技师在首页直接完成高频店务动作
- 组件清单：
  - 组件名：快捷操作卡组
  - 类型（卡片/列表/按钮/分段/弹层）：按钮宫格
  - 字段：操作名、图标、跳转目标
- 数据来源字段：quickActions[]
- 可见条件：首页默认可见
- 排序规则：按使用频率降序
- 文案约束：动词开头，最长 6 个字
- 跳转行为：点击进入对应功能页
- 变更类型：保留（不做调整）

### 区块：`今日待到店区`
- 目标：按时间提醒技师将要接待的顾客，保障提前准备
- 组件清单：
  - 组件名：待到店列表
  - 类型（卡片/列表/按钮/分段/弹层）：列表
  - 字段：顾客名、预约时间、项目、状态、操作按钮
- 数据来源字段：todayArrivalList[]
- 可见条件：首页默认可见
- 排序规则：按预约时间升序（最近将到店优先）
- 文案约束：状态文案统一使用“待到店/已到店/已取消”
- 跳转行为：列表项支持“确认到店”“取消预约”
- 变更类型：调整（明确排序规则与操作闭环）

### 区块：`今日未传服务图区`
- 目标：保留现有服务图上传提醒能力
- 组件清单：
  - 组件名：未传服务图提醒卡
  - 类型（卡片/列表/按钮/分段/弹层）：卡片
  - 字段：待处理数量、查看入口
- 数据来源字段：serviceImagePendingCount
- 可见条件：首页默认可见
- 排序规则：不调整
- 文案约束：保留现有文案
- 跳转行为：保留现有跳转
- 变更类型：保留（不动）

### 区块：`底部栏导航区`
- 目标：统一承接四大模块入口
- 组件清单：
  - 组件名：底部导航栏
  - 类型（卡片/列表/按钮/分段/弹层）：导航
  - 字段：tabName、icon、activeState
- 数据来源字段：bottomTabs[]
- 可见条件：全程固定悬浮
- 排序规则：店务、顾客、应用、私域、数据（固定顺序）
- 文案约束：一级名词，不带描述
- 跳转行为：点击切换到对应页面（数据为占位页）
- 变更类型：调整（新增私域模块tab）

## 5. AI 生成约束（页面级）
- 仅使用纵向单列布局（除明确说明并列卡片）。
- 不允许出现 PRD 未定义组件。
- 每个区块必须有唯一标题，避免语义冲突。
- 首页视觉焦点必须在店务快捷操作区和工资相关指标区。
- 顾客/应用/数据页仅输出入口承接与占位结构，不扩展复杂业务模块。

## 6. 页面：`customer_ops_v1`（顾客运营-新方案-v1）
### 6.1 页面结构（自上而下）
| 区块序号 | 区块名称 | 业务目的 | 是否MVP |
|---:|---|---|---|
| 1 | Header 顶部区 | 明确“顾客运营”页面身份与返回路径 | 是 |
| 2 | 搜索区 | 通过顾客姓名/手机号快速定位顾客 | 是 |
| 3 | 风险顾客基本盘 | 先看基本盘：保有量、新客、流失、休眠 | 是 |
| 4 | 产值规划模块 | 先看目标完成，再看未规划/未达成人数 | 是 |
| 5 | 处方定制模块 | 先看60天未消耗预警，再看到店未到店风险 | 是 |
| 6 | 预约到店模块 | 先看目标结果，再看当日邀约过程数据 | 是 |
| 7 | 客情跟进模块 | 展示三类场景标签并支持模块级跳转 | 是 |
| 8 | 底部导航区 | 店务/顾客/私域/应用/数据切换 | 是 |

### 6.2 区块定义（顾客运营页）
#### 区块：`风险顾客基本盘`
- 目标：首屏先判断顾客盘是否健康，支持后续动作优先级判断。
- 字段：保有量、新客、流失、休眠。
- 排序：固定四项，顺序不变。
- 变更类型：新增（相对首页顾客入口占位）。

#### 区块：`产值规划`
- 目标：让技师快速判断当前产值是否达标以及问题来源。
- 字段：目标值、完成率、排名、未规划人数、有目标未达成人数。
- 排序：结果指标在前（目标/完成率/排名），问题指标在后（未规划/未达成）。
- 变更类型：新增（承接首页迁移的产值进度内容）。

#### 区块：`处方定制`
- 目标：提示项目处方未消耗风险并承接后续邀约处理。
- 字段：60天未消耗项目预警人数、本月有到店计划但未到店人数。
- 排序：60天未消耗优先展示。
- 变更类型：新增。

#### 区块：`预约到店`
- 目标：同步展示预约“结果达标”与“过程推进”。
- 字段：目标值、完成率、排名、今日邀约人数、已邀约人数。
- 排序：结果指标在前，过程指标在后。
- 变更类型：新增。

#### 区块：`客情跟进`
- 目标：聚焦需要优先处理的三类顾客场景。
- 字段：即将流失、重点回访、本月过生日（标签+人数）。
- 排序：按优先级展示（即将流失 > 重点回访 > 本月过生日）。
- 变更类型：新增。
