# 子文档 D：视觉与样式约束（对齐成长值草图基线）

## 1. 画布与基础布局
- 设备宽度：390（移动端）
- 页面内边距：12
- 区块间距：12
- 卡片圆角：12（二级项8）
- 阴影规范：草图阶段不使用重阴影，统一浅背景卡片层次
- 基线来源：`设计/成长值.pen` 中 `店务首页-1比1复刻` 的页面结构和视觉骨架
- 变更原则：仅做增量优化（优先替换局部区块，不整体改版）

## 4. 响应式与兼容
- 最小支持分辨率：390宽（草图评审基线）
- 长文案折行规则：最多两行，超出省略
- 动态数据溢出策略：指标值超长时缩写展示（如 10.2w）

## 5. 可访问性
- 最小对比度：正文与背景对比需可读（遵循默认浅底深字）
- 可点击热区：按钮与tab最小44*44
- 语义化标签要求：每个tab和主按钮必须有唯一命名

## 6. 数据模块占位视觉规则
- 数据模块页仅使用“建设中”占位卡，不出现业务图表。
- 占位卡包含：标题、说明文案、返回首页按钮（可选）。
- 占位态颜色保持中性，不使用强调色作为主视觉。

## 7. Pencil草图指令模板（可直接投喂）
- 页面结构：Header -> 上方数据概览（含销售人数/服务人数）-> 快捷功能（保留）-> 今日待到店（时间排序）-> 今日未传服务图（保留）-> 底部栏
- 底部栏五项：店务、顾客、应用、私域、数据（数据进入占位页）
- 视觉优先级：店务快捷操作区最高，底部栏固定底部
- 占位规则：数据页只显示“建设中”信息，不生成复杂内容
- 输出要求：先复制基线页再做差异修改，并在评审稿标注“保留/调整/新增”区块

## 8. 顾客运营-v1视觉层级要求（仅排版）
- 页面首焦点必须是“风险顾客基本盘”卡片，不允许被其他模块视觉抢占。
- 四个业务模块按“产值规划 -> 处方定制 -> 预约到店 -> 客情跟进”顺序纵向排列。
- 客情跟进模块需同时保留“模块整体跳转位”和“三标签独立点击位”。
- 不允许改动业务文案与字段，仅允许通过字号、留白、边框、对比度强化层级。
