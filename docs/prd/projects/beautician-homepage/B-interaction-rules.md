## 2. 规则定义（必须可执行）

### 规则：`首页焦点优先规则`

- 触发条件（if）：用户进入技师首页
- 判定窗口（when）：页面首屏渲染完成时
- 执行动作（then）：首屏优先展示上方数据概览、快捷功能区、今日待到店区
- 频控/限流：不涉及
- 冲突处理（多规则命中时）：若布局冲突，优先保留店务主区，弱化其他模块首屏信息
- 失败兜底：主区加载失败时展示骨架与重试按钮

### 规则：`今日待到店排序与操作规则`

- 触发条件（if）：存在待到店顾客记录
- 判定窗口（when）：列表渲染和每次状态操作后
- 执行动作（then）：按预约时间升序排序；每条仅提供“确认到店/取消预约”操作
- 频控/限流：单条记录操作后3秒内防重复点击
- 冲突处理（多规则命中时）：已到店/已取消记录置底并禁止再次操作
- 失败兜底：操作失败回滚状态并提示“操作失败，请重试”

### 规则：`底部栏并行入口规则`

- 触发条件（if）：用户点击底部栏任一tab
- 判定窗口（when）：点击后300ms内触发跳转
- 执行动作（then）：进入对应模块入口页（顾客/应用/私域）或占位页（数据）
- 频控/限流：不涉及
- 冲突处理（多规则命中时）：仅激活最后一次点击tab
- 失败兜底：跳转失败返回首页并toast提示

### 规则：`顾客运营模块跳转规则`

- 触发条件（if）：用户进入顾客运营页并点击四大模块标题右侧跳转符
- 判定窗口（when）：点击后300ms内触发跳转
- 执行动作（then）：进入对应模块详情页（产值/处方/预约/跟进）
- 频控/限流：同一模块1秒内防重复点击
- 冲突处理（多规则命中时）：以最后一次点击为准
- 失败兜底：停留当前页并toast提示“暂不可达”

### 规则：`客情跟进双跳转规则`

- 触发条件（if）：用户在客情跟进模块点击“模块标题跳转”或“场景标签”
- 判定窗口（when）：点击触发时
- 执行动作（then）：模块标题跳转到“跟进总览”；场景标签跳转到对应场景列表
- 频控/限流：场景标签1秒内防重复点击
- 冲突处理（多规则命中时）：标题跳转与标签跳转互斥，按实际点击目标执行
- 失败兜底：保持原页并提示“加载失败，请重试”

## 3. 单顾客多问题优先级

- 本轮为首页结构草图，不展开顾客级优先级策略。
- 顾客运营页仅保留“跟进场景入口”占位，不做自动优先级计算。
- 若需沿用成长值优先级，评审后在完整版PRD补充。

## 4. 异常与边界策略


| 场景        | 判定条件            | 页面表现            | 重试机制     | 是否告警 |
| --------- | --------------- | --------------- | -------- | ---- |
| 首页主区接口超时  | 首屏接口超时 > 2s     | 指标区展示骨架和“数据更新中” | 支持点击重试   | 否    |
| 待到店状态更新失败 | 点击确认到店/取消预约接口失败 | 保持原状态并提示失败      | 支持二次点击重试 | 否    |
| 底部栏跳转失败   | 路由未命中或异常        | 保持当前页并提示“暂不可达”  | 自动重试1次   | 否    |
| 数据模块未建设   | 点击数据tab         | 进入占位页展示“建设中”    | 无需重试     | 否    |
