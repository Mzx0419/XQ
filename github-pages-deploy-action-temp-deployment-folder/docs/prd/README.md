# PRD 目录规则（知识库 + 项目产出）

## 目录用途
- `knowledge-base/`：沉淀可复用的 PRD 模板、规范、历史案例与方法论。
- `projects/`：按项目产出新的 PRD 文档，每个项目独立目录管理。

## 现有内容处理原则
- 当前 `docs/prd` 下已有文件默认视为“知识库资产”。
- 后续可按需要逐步迁移到 `knowledge-base/`，不影响当前使用。

## 新任务执行规则
- 每次新建 PRD 前，先确认目标项目目录（必须明确项目名）。
- 若项目目录不存在，先在 `projects/` 下创建该项目文件夹再写入 PRD。
- 同一项目内按模块拆分文档，并提供总览文档链接到子文档。

## 项目目录命名建议
- 使用 `项目名-业务域`，例如：`beautician-growth`、`crm-retention`。
- 避免中文空格和特殊符号，统一小写加中划线。
