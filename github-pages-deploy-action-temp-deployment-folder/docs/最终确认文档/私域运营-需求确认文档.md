# 私域运营需求确认文档

## 1. 整体目标、价值、使用人群、大致场景

### 1.1 整体目标

- 通过“技师管理-顾客维度”的筛选与分层能力，快速识别高价值与高风险顾客。
- 通过“详情页提醒 + 任务引导 + 优惠券临期跟进”形成连续运营动作，提升顾客留存活跃度。

### 1.2 功能价值（老板视角）

- 经营价值：把“顾客是否活跃”从经验判断变成可筛选、可追踪、可复盘的数据闭环。
- 留存价值：通过任务参与与到店行为双维驱动留存，其中到店行为权重更高。
- 转化价值：通过临期优惠券预约提醒，提升顾客复访与到店几率。
- 管理价值：同一页面完成“识别-触达-跟进-复盘”，减少跨页面决策成本。

### 1.3 使用人群

- 核心使用人群：店长、运营负责人、技师管理者。
- 关注结果人群：老板/业务负责人（看留存、到店、复访结果）。

### 1.4 大致场景

- 在私域运营总览页按“徽章类型、任务类型、成长值、等级”筛选顾客。
- 点击顾客进入详情页，判断成长值问题与任务完成情况。
- 通过“去提醒”分享微信小程序或任务链接给顾客，推动顾客完成任务或到店。
- 通过优惠券到期信息，提前触发预约到店跟进。

### 1.5 版本边界（本次确认会）

- 本次范围（V1）：筛选识别、详情总览、提醒分享、任务信息、优惠券到期提示、任务明细。
- 非本次范围（V2+）：智能推荐最优触达文案、自动化分群策略引擎、A/B策略自动投放。

## 2. 按页面与流程拆分（映射6条核心诉求）

### 2.1 流程1：顾客维度筛选识别（对应诉求1）

- 对应页面：私域运营（节点 `VTfvR`）。
- 对应模块：筛选区（徽章类型、任务类型、成长值等级）+ 顾客卡片列表。
- 业务流程：
  1. 管理者按筛选条件组合筛选顾客。
  2. 系统返回符合条件的顾客卡片及任务完成状态。
  3. 管理者优先处理高风险或高潜顾客。
- 业务价值：
  - 从“全量人群粗放运营”转为“分层人群精准运营”。
  - 提升触达效率与运营投入产出比。

### 2.2 流程2：进入详情并提醒顾客参与活动（对应诉求2）

- 对应页面：私域运营详情页（节点 `DEb7B`）。
- 对应模块：任务卡片中的“去提醒”按钮。
- 业务流程：
  1. 点击某顾客进入详情页，定位成长值缺口与任务状态。
  2. 点击“去提醒”，分享微信小程序到顾客微信。
  3. 顾客收到提醒后进入活动页参与任务。
- 业务价值：
  - 把“发现问题”直接转成“触达动作”，缩短运营闭环路径。
  - 让触达动作可追踪，便于复盘触达效果。

### 2.3 流程3：成长值总览督促兑换（对应诉求3）

- 对应页面：私域运营详情页。
- 对应模块：成长值卡（当前成长值、等级、升级进度、累计获得、累计兑换）。
- 业务流程：
  1. 查看顾客当前等级与距离下一等级差值。
  2. 查看累计获得与累计兑换判断成长值使用习惯。
  3. 引导顾客兑换优惠券，减少成长值沉淀浪费。
- 业务价值：
  - 提升成长值体系的实际使用率。
  - 通过兑换行为拉动后续到店与消费意愿。

### 2.4 流程4：任务信息识别偏好并支持直达（对应诉求4）

- 对应页面：私域运营详情页。
- 对应模块：任务类型筛选 + 任务模块（活跃/交互/链接/分享/裂变）。
- 业务流程：
  1. 通过任务模块查看顾客偏好任务类型。
  2. 选择更匹配的任务链接分享给顾客。
  3. 顾客点击链接可直接进入任务执行。
- 业务价值：
  - 从“统一任务投放”升级为“偏好型任务匹配”。
  - 提升任务转化率与完成率。

### 2.5 流程5：优惠券到期跟进预约到店（对应诉求5）

- 对应页面：私域运营详情页。
- 对应模块：优惠券模块（未使用/已兑换、7/30/45天到期分层、到期清单）。
- 业务流程：
  1. 查看顾客已兑换券与临期分布。
  2. 优先跟进7天内到期券顾客。
  3. 触发顾客预约到店，提升券核销与复访。
- 业务价值：
  - 把“券到期风险”转为“预约到店机会”。
  - 直接提升到店概率与复访机会。

### 2.6 流程6：任务明细复盘近期行为（对应诉求6）

- 对应页面：私域运营详情页。
- 对应模块：任务明细（时间、任务事件、成长值增减）。
- 业务流程：
  1. 查看顾客最近完成任务与成长值变化。
  2. 判断顾客行为趋势（持续活跃、短期沉默、兑换倾向）。
  3. 输出下一步运营动作（提醒、任务推送、预约引导）。
- 业务价值：
  - 让运营动作有依据，减少无效触达。
  - 支撑管理者按顾客状态做差异化策略。

## 3. 价值定位与指标口径（已确认）

### 3.1 北极星指标

- 北极星指标：顾客留存活跃度（7日）。
- 活跃判定：任务与到店都计入活跃，但到店行为权重更高。

### 3.2 指标模型（V1建议）

- 7日加权活跃分 = `到店次数 x 0.7 + 任务完成次数 x 0.3`
- 7日活跃顾客判定：7日加权活跃分 >= 1 记为活跃顾客。
- 7日活跃顾客率 = `7日活跃顾客数 / 纳入运营顾客总数`

说明：V1先用可解释的简化模型，后续可按实际运营效果调整权重或阈值。

### 3.3 一级与二级指标

- 一级指标（结果）：
  - 7日活跃顾客率（核心）
  - 7日到店预约率（关键支撑）
- 二级指标（过程）：
  - 提醒发送率（有风险顾客中被提醒占比）
  - 提醒点击率（提醒后进入小程序/链接占比）
  - 任务完成率（被触达后任务完成占比）
  - 临期券转预约率（临期券顾客中形成预约占比）

### 3.4 价值链路

识别风险顾客 -> 发起精准提醒 -> 任务参与/到店行为 -> 活跃度提升 -> 留存与复访提升

## 4. 验收标准（需求确认口径）

- 筛选准确：按徽章类型、任务类型、成长值等级筛选后，顾客集合与条件一致。
- 链路可达：从总览页可进入详情页；从详情页可触发“去提醒”并完成分享。
- 状态可读：成长值进度、任务状态、优惠券临期状态、任务明细可被快速理解。
- 动作可追踪：提醒发送、提醒点击、任务完成、预约到店可被记录统计。
- 数据可解释：活跃判定口径与权重规则可对老板清晰说明。

---

## 5. 风险与边界

- 口径风险：若不同团队对“活跃”定义不一致，可能导致目标理解偏差。
- 触达风险：提醒频次过高可能引起顾客反感，需后续补充触达频控策略。
- 数据风险：数据延迟会影响“当下判断”，需标注数据更新时间。
- 边界约束：V1不包含自动化策略引擎与智能文案生成，先保证人工策略闭环可跑通。

## 6. 本期必须做 / 可后续优化

### 6.1 本期必须做（V1）

- 顾客维度筛选（徽章类型、任务类型、成长值、等级）。
- 顾客详情页的成长值、任务信息、优惠券、任务明细四大模块。
- “去提醒”分享动作打通（微信小程序/任务链接触达）。
- 留存活跃度指标看板（7日加权活跃口径）。

### 6.2 可后续优化（V2+）

- 自动推荐“下一条最优触达动作”（任务/优惠券/预约优先级）。
- 基于顾客标签的自动分群与自动化触达编排。
- 任务文案与提醒内容A/B测试能力。
- 权重模型动态优化（按门店、客群、阶段自动调参）。
