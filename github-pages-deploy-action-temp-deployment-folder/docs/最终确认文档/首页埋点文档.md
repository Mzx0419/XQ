# 店务埋点文档编写计划（按角色大类分块）

## 文档目标

按**角色大类**（技师、前台等）分块，为各角色下的页面产出**埋点说明文档**，每条埋点采用你提供的统一格式：

- 登录账号
- 点击时间
- 点击数量
- 功能模块
- 功能按钮
- 设备类型：安卓 / IOS
- 版本号：App 版本号

## 文档结构建议

1. **文档头部（全局）**
  - 简要说明：本文档按**角色大类**分块描述店务相关页面的点击/曝光埋点，包含**技师**（技师首页、订单类型页）与**前台**（前台首页）。
2. **角色大类一：技师**
  - **页面 1：技师首页 / 仪表盘**
    - 页面名称：技师首页 / 仪表盘
    - 页面 ID：如 `tech_home_page` 或 `tech_dashboard`
    - **埋点列表**（按界面区域分组）：为每个可点击/可曝光的「功能按钮」写一条埋点，同一模块下可共用「功能模块」名称。

    | 区域    | 功能模块   | 功能按钮（埋点条目）                      |
    | ----- | ------ | ------------------------------- |
    | 顶部导航  | 全局导航   | 侧边栏菜单、消息通知入口                    |
    | 业绩概览  | 业绩概览   | 销售业绩卡片、消耗业绩卡片                   |
    | 快捷操作  | 业务操作   | 开单制单入口（）、扫码核销入口、充值入口、预约入口、预收款入口 |
    | 今日待到店 | 待到店管理  | 待到店列表入口/查看更多、预约详情卡片、取消预约、到店确认   |
    | 服务状态  | 服务任务概览 | 今日未传服务图列表入口、今日服务单未评价列表入口        |
    | 当月产值  | 产值进度   | 产值进度详情入口、产值进度图表                 |
    | 底部导航  | 底部导航   | 店务、顾客、应用、数据                     |

  - **页面 2：订单类型（点击「开单制单」后进入）**
    - 页面名称：订单类型
    - 页面 ID：如 `order_type_select` 或 `create_order_type`
    - 入口：技师首页 → 开单制单入口
    - **埋点列表**：

    | 区域   | 功能模块   | 功能按钮（埋点条目）               |
    | ---- | ------ | ------------------------ |
    | 制单类型 | 订单类型选择 | 销售制单、服务制单、回收制单、未购制单、订金制单 |

3. **角色大类二：前台**
  - **页面：前台首页**
    - 页面名称：前台首页
    - 页面 ID：如 `front_desk_home_page` 或 `front_desk_dashboard`
    - **埋点列表**（按界面区域分组）：

    | 区域     | 功能模块      | 功能按钮（埋点条目）                                       |
    | ------ | --------- | ------------------------------------------------ |
    | 顶部导航栏  | 页面导航/用户入口 | 菜单入口、消息中心入口、个人中心入口                               |
    | 数据概览卡片 | 业绩数据概览    | 实收业绩入口、上报销售入口、实耗业绩入口、上报消耗入口、进店人数入口、服务人数入口、付款人数入口 |
    | 快捷功能区  | 业务操作      | 订单结算入口、业绩复核入口、核销入口、退款入口、预收款入口                    |
    | 最近消息区  | 消息通知      | 最近消息区入口（整区点击）                                    |

4. **单条埋点格式（示例）**
  每条埋点在文档中写成一致格式，便于直接给数据/前端使用。其中「登录账号」「点击时间」「点击数量」在文档中可写为**占位说明**（实际由上报时传入），例如：

```
登录账号：（上报时传入当前用户账号）
点击时间：（上报时传入，如 2026.1.12）
点击数量：1
功能模块：开单制单
功能按钮：开单制单入口
设备类型：安卓 / IOS（上报时传入）
版本号：（上报时传入，如 1.2.0）
```
